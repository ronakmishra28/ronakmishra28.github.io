Home Lab · WAF · Web Security
SafeLine WAF — Web Attack Detection & Blocking Lab
Read Full Report GitHub

Deployed a fully isolated home lab using DVWA (Damn Vulnerable Web Application) on an Ubuntu Server VM and an Kali Linux attacker VM — both on bridged networking to simulate a real threat environment. Executed live web attacks including SQL Injection, Cross-Site Scripting (XSS), and HTTP Flood against the unprotected application to confirm exploitability, then deployed SafeLine WAF (Docker-based, open-source) as a reverse proxy in front of DVWA.

Configured security rules including rate limiting, IP blocking, authentication enforcement, and SQL injection/XSS detection in Balance mode. Verified that every attack was blocked — confirmed via 403 responses and real-time log entries in the SafeLine dashboard showing source IP, matched rule, payload, and timestamp. The log review workflow directly mirrors what SOC analysts do when investigating suspicious web traffic.
